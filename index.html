Safire SMS Remote Controller

The application lets Safire heater owners control the heaters at their homes, cabins, boats, etc. easily and remotely. Based on the intuitive and easy-to-use interface, the application generates SMS messages for the user to send to their heater's SMS unit. The application can no longer directly send SMS messages directly from the app due to the changes in Google Play's terms of service. However, the application opens up the premade SMS message within the user's SMS software which then only require a tap of the Send button. The application is multilingual, currently supporting Finnish and English.

RPG Dice Roller "Dicey Dicing"

While physical dice are a lovely thing, sometimes they cannot beat the flexibility of digital dice. For example, you do not often carry around dozens of each dice type everyone you go, so it is quite useful to have a digital backup. Also, using an application is somewhat quicker since it also does most of the math for you.

Current features:

  • Quick and easy dice rolling and calculation for many RPG and general uses!
  • 9 types of dice with adjustable dice amount
  • Animations that add to the application's responsiveness
  • Quirky little sounds on button presses, though the sound can be turned off from the menu
  • The ability to see individual dice roll scores (up to 8 dice)
  • Made with Android Studio / Java

Potential upcoming features:

  • Roll history
  • Adding modifiers to rolls, such as a roll of 2d8 + 3
  • Improvements to graphics and sound

Custom FPS Controller with Parkour Mechanics

In my free time I am developing a custom FPS controller I've made for Unity. The idea behind the controller is having the character still be a physics-based object (reacting to forceful effects such as explosions) while still having tight and responsive controls. Additionally, I've decided to expand upon its code with dynamic parkour mechanics: This way I would not have to - for example - specify which ledges the player could grab and climb.

Current features:

  • Snappy physics-based basic FPS controls with jumping, crouching
  • Dynamic ledge climbing system: No areas need to be manually marked as climbable, as the angle and space above a ledge determines whether it can be climbed or not.
  • Context-sensitive controls: For example hanging on a ledge has its own controls that are separate from the basic ones
  • Made with Unity / C#

Planned upcoming features:

  • Sideways movement along ledges
  • Sliding
  • Dynamic animations based on climbed terrain
  • Guns

Tabletop RPG Manager (offline)

Having used and tested several solutions (Roll20, Fantasy Grounds, PowerPoint, image manipulation programs...) to manage my tabletop RPG games as the Game Master, I've found that none of the available tools seem to provide all of the most the necessary functionalities I need in my games. They lack, for example, a very quick dynamic system for generating props combined with the ability to both snap on and off a grid system. Some systems might also require you to always be connected to the internet even if you would not need other players to join any online session. Therefore, I've started creating my own system for a grid-based tabletop RPG manager which aims to have all of these functionalities.

Current features:

  • Dynamic grid system (snaps on/off grid at will) with smooth drag 'n' drop
  • Quick spawning system for PCs/NPCs and AoE effects
  • Distance calculation in both unit systems
  • Simple save system for 1 view
  • Simple dynamic drawing on the map/grid
  • Does not require an internet connection
  • Made with Unity / C#

Planned upcoming features:

  • Partly duplicated screen: The Game Master will have their own screen and can dynamically hide and show objects in the players' view
  • Easy-to-use save slot system for different views/scenes
  • Quick spawning system for all moveable objects
  • Tagging objects as immoveable
  • Minor network features: Players could mark objects, points or paths on their devices and have those markings visible for the Game Master (optional)
  • Dynamically uploading new prop images to the program

RPG Initiative and Character tracker "Trackey Tracking"

Just a small hobby project to make it slightly quicker for me to manage especially combat encounters in tabletop RPGs. By making the app run entirely in a browser, it is simply easier for me to separate my more static notes from much more dynamic character statuses and situations.

Current features:

  • Adding characters with health points, armor class etc.
  • Saving and loading the characters and all their data. Saving and loading can be done either with the browser cache or with JSON files.
  • Most non-character specific commands can be handled via the implemented keyboard input system.
  • Currently compatible with at least Dungeons & Dragons 3.5
  • Made with HTML, CSS, JS (vanilla) / Electron

Planned upcoming features:

  • Improved graphics
  • Better error correction: Every single character data point should be editable soon.
  • Extensions to character data: Inventory, spells etc. Currently the extra data can only be set into a regular text field.
